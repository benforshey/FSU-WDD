Categories of Web Development (Go to the Top)

There's no hard and fast definition for these terms, but I loosely define the categories of web development as follows:

Front End
A developer who primarily works with HTML, CSS, and JavaScript.
Back End
A developer who primarily works with server–side languages like PHP, CSS, Ruby, Python, or Node.js.
Full Stack
A developer who is knowledgeable in both areas, equally balanced and skillful in both.

I have the blessing of being able to go to college for a new career, and though I'm not yet skilled in either the front end or the back end of web development, I’m immensely enjoying the study of both.

Back End

I migrated my server from Apache to Nginx a few days ago. In my previous setup I was running a LAMP stack on a VPS. I wasn’t happy with my theoretical “simultaneous users” limit I was running into under stress testing (something to do with a thread–driven architecture).

I switched to Nginx, which took a bit of configuring to get working. I’m really satisfied with the event–based reactor pattern that the program uses. I don’t understand too much of the computer science stuff, but I do understand that I'm doing much better serving HTTP requests under heavy load. Not only that, but it's pretty easy to configure for gzip and static resource caching!

I think the best part about running my own server is the experience gained. There‘s nothing like learning by doing! It‘s pretty wild to think that I can host webpages from a remote computer—all configured through the command line.
